NAME:
   uipath orchestrator process-schedules post - Creates a new process schedule.

USAGE:
   uipath orchestrator process-schedules post [command options] [arguments...]

DESCRIPTION:
   OAuth required scopes: OR.Jobs or OR.Jobs.Write.

   Required permissions: Schedules.Create.

OPTIONS:
   --folder-id integer (required)
      Folder/OrganizationUnit Id

   --name string (required)
      The name of the schedule.

   --time-zone-id string (required)
      The timezone under which the schedule will run.

   --activate-on-job-complete boolean
      Flag for activating the queue trigger on each completed job.

   --alert-pending-expression string

   --alert-running-expression string

   --calendar-id integer
      The id of the calendar that a process schedule uses.

   --calendar-key string

   --calendar-name string
      The name of the calendar.

   --consecutive-job-failures-threshold integer

   --description string
      Description of process schedule.

   --enabled boolean
      Specifies if the schedule is active or not.

   --environment-id string
      The Id of the environment associated with the schedule.

   --environment-name string
      The name of the environment associated with the schedule.

   --executor-robots object (multiple)
      The collection of specific robots selected to be targeted by the current schedule. This collection must be empty if the start strategy is not 0 (specific robots).

      Example:
         Description=string; Id=integer; MachineName=string; Name=string

   --external-job-key string
      The unique identifier of the external job associated with the jobs generated by this schedule. A key is generated for each group of jobs triggered by this schedule.

   --external-job-key-scheduler string
      The unique identifier of the external background task schedule associated with the background tasks generated by this schedule. A key is generated for each batch of background tasks triggered by this schedule.

   --id integer

   --input-arguments string
      Input parameters that will be passed to each job created by this schedule.

   --is-connected boolean
      Whether this process trigger represents a connected trigger defined in package bindings

   --items-activation-threshold integer
      The minimum number of queue items that should trigger the process activation.

   --items-per-job-activation-target integer
      The target ratio between the number of queue items and the Total number of jobs created by a process.

   --job-failures-grace-period-in-hours integer

   --job-priority string
      The execution priority. If null, it will start jobs with the release's priority.

      Allowed values:
      - Low
      - Normal
      - High

   --key string
      The Key of the process schedule.

   --kill-process-expression string
      Grace period (in seconds) for soft stop. If a process doesn't stop after this amount, it will be killed

   --machine-robots object (multiple)
      The machine robots.

      Example:
         MachineId=integer; MachineName=string; RobotId=integer; RobotUserName=string; SessionId=integer; SessionName=string

   --max-jobs-for-activation integer
      The maximum number of jobs that a process can create as result of a Queue driven activation.

   --package-name string
      The name of the package to be triggered with the schedule.

   --queue-definition-id integer
      The Id of the queue that uses this schedule for activation (trigger jobs when new queue items are added)

   --queue-definition-name string
      The Name of the queue that uses this schedule for activation (trigger jobs when new queue items are added)

   --release-id integer
      The Id of the process associated with the schedule.

   --release-key string
      The unique key of the process associated with the schedule.

   --release-name string
      The name of the process associated with the schedule.

   --resume-on-same-context boolean
      Flag for honoring initial machine and robot choice upon resumption of created jobs, if jobs are suspended. <br />
      If set, the jobs will resume on the same robot-machine pair on which they initially ran.

   --run-as-me boolean

   --runtime-type string
      The runtime type of the robot

      Allowed values:
      - NonProduction
      - Attended
      - Unattended
      - Development
      - Studio
      - RpaDeveloper
      - StudioX
      - CitizenDeveloper
      - Headless
      - StudioPro
      - RpaDeveloperPro
      - TestAutomation
      - AutomationCloud
      - Serverless
      - AutomationKit
      - ServerlessTestAutomation
      - AutomationCloudTestAutomation
      - AttendedStudioWeb
      - Hosting
      - AssistantWeb
      - ProcessOrchestration
      - AgentService

   --specific-priority-value integer
      Value for more granular control over execution priority.

   --start-process-cron string
      The start cron expression of the schedule.

   --start-process-cron-details string
      Various details that can be associated to the time period expression of the schedule.

   --start-process-cron-summary string
      Human readable form of cron expression of the schedule.

   --start-process-next-occurrence string
      The date and time when the associated process will be run next.

   --start-strategy integer
      States which robots from the environment are being run by the schedule.

   --stop-process-date string

   --stop-process-expression string
      Number of seconds after which a running process will be stopped.

   --stop-strategy string
      The way a running process is stopped.

      Allowed values:
      - SoftStop
      - Kill

   --tags object (multiple)

      Example:
         DisplayName=string; DisplayValue=string; Name=string; Value=string

   --time-zone-iana string
      The timezone under which the schedule will run in Iana Standard.

   --use-calendar boolean
      DEPRECATED. Specify whether the schedule uses any calendar.
